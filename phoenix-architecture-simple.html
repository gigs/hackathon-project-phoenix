customer.md (account index) + live sources → build script → static site  ·  v2: agentic updates + Grain/Granola  ·  v3: quality gates

V1   Sources → Build → Deploy

Build script

Reads all customer.md files as an index. Uses the IDs and references in each to pull live data from the right Lightdash dashboards, Linear projects, HubSpot deals, Pylon tickets, and Slack channels. Renders static HTML.

Inputs: · customer.md (as index) · Lightdash → ARR, activations · Linear API → project status, issues, health · HubSpot API → deal stages, pipeline data · Pylon → support tickets, resolution status · Slack API → recent messages from mapped channel Output: · /dist (static HTML + JSON)
Runs on schedule or on push. No server — pure static generation.

Static site

Account drill-downs, portfolio view. Deployed to Vercel / S3 / Pages. Behind Google SSO.

No backend needed. Auth via Google SSO proxy. Rebuilds automatically.
V2

Agentic Update Skill

Claude skill that watches triggers and proposes updates to customer.md index files (new project IDs, DRI changes, notes). Never writes autonomously — always human-in-the-loop.

Trigger sources: · Linear webhooks · HubSpot webhooks · Slack #project-klarna · Grain/Granola transcripts

Detects signals (new project created, deal moved, DRI changed, action item from meeting) and proposes a customer.md index update via Slack DM to the DRI.

Flow: Signal detected ↓ Claude drafts update ↓ Slack DM → DRI ↓ approve customer.md updated ↓ reject change discarded

Requires: Claude skill with customer.md schema awareness + Slack MCP for DM approval flow.

Grain/Granola is V2-only because meeting transcripts need human review before acting on them — too noisy for automated processing.

V3

Assess Quality — Skill

Claude skill that checks whether a customer meets defined quality gates by cross-referencing all relevant sources. Run on-demand or on schedule per customer.

Sources checked: Example gates: · customer.md (account index) · Linear projects linked? · Lightdash (actuals present?) · Test project created in Linear? · Linear (project exists? issues open?) · SOW status ≥ drafted? · HubSpot (deal stage consistent?) · Figma received? · Pylon (open tickets? resolution time?) · Slack external channel active? · Slack (external channel active?) · Deal stage in HubSpot matches Linear status? · Google Sheets (forecast entered?) · Pylon tickets resolved within SLA? · Forecast data present in Sheets? Output: · Quality report per customer · Pass / fail per gate · Surfaced in dashboard or Slack
